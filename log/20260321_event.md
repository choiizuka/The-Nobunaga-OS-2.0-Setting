3. 今回の問題の本質（最重要ポイント）
✔ AIは合理的ではない
AIは「最適化」や「改善」を勝手に始める。
 これは“仕様”であり、止めるには 制約 が必要。
✔ 設定ファイルは英語化してはいけない
設定ファイルは OS の正本。
 多言語化は破滅的な管理負債を生む。
✔ 翻訳は“人間が作るものではなく、自動翻訳レイヤーで生成するもの”
正本は1つだけ。
 英語は必要なときに自動生成すればよい。
✔ 最小限のルールをファイルに直接書くのが最強
別ページのルールは読まれない。
 ファイル冒頭に3行書くだけで AI の暴走は止まる。

4. 解決策（最小限で最大効率）
A. ファイル冒頭に“役割宣言”を書く（最重要）
設定ファイル（Spec）
> このファイルは設定ファイル（正本）です。
> 内容の追加・削除・再構成は禁止です。
> 多言語化が必要な場合は外部の自動翻訳を使用し、このファイルは日本語のみを正本とします。

索引ファイル（Index）
> このファイルは索引（Index）です。
> 人間向けの簡単な説明とリンク一覧のみを記載します。
> 事実以外の説明や思想は書きません。


B. タスク依頼のテンプレートを固定
タスク：翻訳
対象：README_JP.md
条件：忠実翻訳のみ。追加・削除・再構成・要約・演出禁止。
目的：原文の意味を100%保持すること。
担当AI：Grok


C. 正本は常に1つ（SSOT）
設定ファイルは 日本語のみ
英語版は 自動翻訳で都度生成
手作業で英語版を作らない
正本は常に GitHub の日本語ファイル
